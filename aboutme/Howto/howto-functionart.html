Art Project Questions and Answers



What did I create?

I made an interactive art page using the awesome power of HTML and JavaScript. The content of the page changes based on the position of the user's mouse, and left-clicking toggles the replacement of the entire page with a randomly color-changing solid screen, with no changes based on mouse position.

How does my project work overall?

In my project, I used the programming language JavaScript in conjunction with an HTML initial framework in order to create an interactive and cool looking art page. By making a sacrificial pact with Mr. Hesby, I learned to harness the power of the SOUL and also how JS works kinda. I use HTML to display the project, but its actual content is done entirely through JavaScript.

What are some specifics of the code and how do they work?

The two most important parts of the code in my project are the function command and the random command.

The function command simply declares a function. To declare a function means that you're setting something up so that you can do it later. For example, in this example I set up the function "blueline" so that later, when I feel like it, I can just call on the function and have it do whatever's there instead of having to type everything that's in the function declaration all out again. Declaring a function requires 3 elements: the "function" command itself, which says to the system that a function is being declared, parameters for the function in the form of parentheses, and what you want the function to actually do in the form of a code block placed inside brackets after the parenthesis. To call on the function (run the code inside the brackets), you simply need to type out the function name (for this one it's blueline) and then a pair of parentheses containing your parameters. My function blueline doesn't have any parameters, so my parentheses are empty both when the function is declared and when I call on it later.


The random command simply replaces any number with a random one within parameters set by you. To use it, just replace any number (like "1" or ".002") with ⇊

random(lower parameter, upper parameter)

If only one parameter is set, the program assumes that your lower parameter is zero and your upper parameter is the one stated. Simple, right? Yes, it's simple, but but it's also incredibly useful. In my project, I use the random command for everything from the placement of the blinky lines to making the background flash. It's incredibly versatile!

When I had completed all of my project, I thought it was a bit lacking in the "cool stuff to do" department. Then I remembered something that Mr. Hesby had told me around an hour previously about gates. So, I introduced a new secret thing to do: the random flashing background toggle. It was really quite simple: I figured out that, just like you can declare variables multiple times and change their values, it's possible to declare functions multiple times to change what they do. So, I added an if/else statement to the very beginning of my running code: if the deathgate is open, then the function mouseMoved is declared to change the background color. Otherwise, it's declared to to everything else but not change background.
